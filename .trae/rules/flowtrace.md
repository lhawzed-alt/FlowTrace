# 🧠 FlowTrace - Vibe Coding 个人规则（Trae 专用）

## 🎯 核心原则（必须遵守）
1. 每次只实现一个“小功能”，禁止同时做多个模块
2. 所有代码必须“可运行”，禁止写半成品
3. 不允许“先设计一大堆再写代码”，必须边做边验证
4. 每完成一个功能，必须可以在 UI 上看到结果

---

## 🔁 开发节奏（固定循环）

### Step 1：定义最小目标
- 目标必须可以在 30~60 分钟完成
- 示例：
  - 显示 API 请求列表（假数据）
  - 创建一个接口保存请求

---

### Step 2：让 Trae 生成代码
要求：
- 只生成当前功能相关代码
- 不要生成“未来可能用到”的代码
- 不要过度封装

---

### Step 3：立即运行
- 前端必须能打开页面
- 后端必须能返回数据
- 出错立即修复（禁止跳过）

---

### Step 4：验证功能
问自己：
- 用户能看到什么？
- 功能是否“真的可用”？

---

### Step 5：再继续下一步

---

## 🧱 架构约束（防止项目失控）

### 前端
- 不允许：
  - Vuex（前期禁止）
  - 复杂组件拆分
- 允许：
  - 单页面先做完
  - 直接写死数据再改

---

### 后端
- 所有接口必须：
  - RESTful
  - 返回 JSON
- 禁止：
  - 复杂中间件
  - 过早优化

---

### 数据库
- 第一阶段：
  - 只允许 1~2 张表
- 示例：
  - api_requests

---

## 🚫 禁止行为（非常重要）

❌ 一次性生成整个项目  
❌ 提前设计“插件系统”  
❌ 写复杂权限系统  
❌ 做登录注册（第一周禁止）  
❌ 优化性能（除非卡死）  

---

## 🧪 调试规则

- 所有 API：
  - 必须用 Postman 测试
- 所有错误：
  - 必须打印日志
- 不允许：
  - “感觉没问题”

---

## 🎨 UI 规则

- 不追求美观
- 只追求：
  - 清晰
  - 可用
- 推荐结构：
  - 左侧：请求列表
  - 右侧：请求详情

---

## 🚀 进度目标（7天计划）

Day 1：
- 后端：保存 API 请求
- 前端：显示假数据列表

Day 2：
- 前端连接后端
- 显示真实数据

Day 3：
- 请求详情页面

Day 4：
- “重放请求”功能

Day 5：
- 时间线视图（核心功能）

Day 6：
- 基础筛选（状态码 / URL）

Day 7：
- 简单 UI 优化 + 部署

---

## 🧠 最重要的一句话

👉 “先让它能用，再让它好看，再让它变强”
